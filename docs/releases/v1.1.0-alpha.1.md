## Security Notes

- Secret handling: public docs use placeholders only; usable credentials must
  remain outside the repository.
- Webhook posture: production WhatsApp webhooks require HTTPS, unique verify
  tokens, and `META_APP_SECRET` signature validation.
- Token/encryption posture: WhatsApp and integration tokens must be stored
  encrypted at rest with installation-specific secrets.
- Vulnerability disclosure notes: report vulnerabilities privately through
  `SECURITY.md`; do not open public issues containing exploit details or
  secrets.
- Repository controls: vulnerability alerts, Dependabot security updates,
  secret scanning, secret scanning push protection, CI, labels, and protected
  `main` are enabled.
