## Upgrade Notes

- Migrations: No database migration.
- Environment variables: No new required environment variables.
- Manual steps: Existing self-hosted users should review
  `docs/deployment/environment.md`, `docs/deployment/upgrades.md`, and
  `SECURITY.md` before adopting this alpha.
- Backup or restore requirements: No new backup format or restore procedure.
  Continue to back up PostgreSQL and persistent application volumes before
  upgrading.
- Rollback limitations: This alpha includes dependency updates. If a deployment
  needs rollback, redeploy the previous known-good source commit or image and
  restore from backup if application data was changed after upgrade.

## Known Limitations

- This is an alpha prerelease, not a stable production release.
- GitHub currently reports default-branch dependency vulnerabilities. Details
  should be reviewed through private Dependabot alerts by an authorized
  maintainer; do not copy private alert data into public release notes.
- The `esbuild` `0.28.1` Dependabot update was deferred because required checks
  failed in pull request `#24`.
- No production deployment, customer deployment, production migration, DNS
  change, credential change, billing change, or real WhatsApp send is included
  in this prerelease.

## Security Notes

- Secret handling: no repository secrets or usable credentials are introduced;
  examples remain placeholders.
- Webhook posture: documentation continues to require HTTPS, unique verify
  tokens, and Meta signature validation for production WhatsApp webhooks.
- Token/encryption posture: documentation continues to require encrypted token
  storage and installation-specific secrets.
- Dependency or vulnerability notes: Next.js, Drizzle ORM, Vitest, direct
  PostCSS, and nested PostCSS versions were updated by Dependabot.
  Default-branch Dependabot vulnerability alerts remain a separate private
  security review item.
- Vulnerability disclosure notes: use `SECURITY.md`; do not open public issues
  containing exploit details, secrets, customer data, or private logs.
