# Roadmap

Reservas CRM is a self-hosted WhatsApp CRM for customer conversations,
AI-assisted operations, and reservation workflows. The roadmap is public
direction, not a delivery promise. Release scope is finalized through issues,
specs, pull requests, changelog entries, and release evidence.

## Now

- Keep the public repository clean, professional, and safe to clone.
- Establish pull-request CI, branch protection, and maintainer review flow.
- Document changelog, release evidence, repository labels, and maintenance
  rhythm.
- Improve public deployment guidance for Docker Compose and self-hosted
  operators.

## Next

- Add reproducible fresh-install smoke testing.
- Expand deployment documentation for HTTPS, WhatsApp webhooks, backups,
  restores, and upgrades.
- Keep screenshots current with synthetic data from the maintained Reservas CRM
  UI.
- Prepare `v1.1.0-alpha.1` as the first prerelease after the public foundation
  work is reviewed.

## Later

- Strengthen reservation workflows, resource availability, holds, reminders,
  and operational reporting.
- Improve public maintenance automation once manual repository hygiene and
  release evidence are stable.
- Evaluate additional public integrations only when they preserve CRM authority,
  customer isolation, and self-hosted deployment boundaries.

## Not On The Public Roadmap

- Private backend strategy, customer-specific deployment details, production
  credentials, private customer data, or internal business notes.
- Moving authoritative reservation decisions into prompts, n8n workflows, or
  external automation.
